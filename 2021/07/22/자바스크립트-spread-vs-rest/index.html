자바스크립트 : spread vs rest

Posted on 2021-07-22

안녕하세요, 오늘은 자바스크립트의 연산자 중 spread와 rest에 대해서 말해보고자 합니다! spread와 rest 모두 편리하게 사용이 가능하여 알아두시면 좋은 연산자입니다.

함께 공부하러가봅시다!

spread

  1. spread는 기존에 있던 배열을 참고해서 새로운 속성을 추가할 때 많이 사용됩니다. 이는 배열과 객체 모두에 사용이 가능합니다. 예를 들어, ‘abcdeft’라는 문자열을 배열에 추가하고 싶다면 기존 배열/객체에 ...(이름)을 추가해주면 됩니다.

또한, 여러번 값을 가지고오는 것 또한 어렵지않게 가지고올 수 있습니다. 그냥 여러번 적어주면 (…spread를) 됩니다!

  1. spread는 또한 배열을 복사해올수 있습니다. 완전히 똑같은 값을 복사해올수도 있고 새로운 값을 추가한채로 복사해올수도 있습니다.
  1. 배열의 concat 문법을 대체하여 사용이 가능합니다. 중간에 값을 가지고온채로 합칠수있기 때문에 concat보다 더 유동성있게 사용이가능하다는 장점이 있습니다.

rest

spread와 rest의 차이는 spread는 값을 퍼뜨린다면 rest는 값을 모아옵니다. 조금 헷갈리시나요?

rest는 주로 객체, 배열, 함수의 파라미터에서 사용합니다. 단, spread와 두드러지는 차이점은 앞이나 중간에 사용할 수 없고 항상 맨 마지막에 …rest를 사용해주어야한다는 것입니다. 파라미터에 사용되는 것과 맨 마지막에 사용된다면 rest라는 것을 빠르게 파악가능할 것입니다.

자, 이렇게 spread와 rest에 대해서 알아보았는데 어떠신가요? rest는 몇 개가 들어올지 모르는 인자값을 대체해 사용하능할 때 많이 활용된답니다.

오늘도 즐거운 하루 되세요!
