자바스크립트 : reduce 함수란?

Posted on 2021-07-21

안녕하세요, 오늘은 자바스크립트의 배열 함수인 reduce에 대해서 말해보고자해요. 처음에 reduce를 봤을때는 많이 혼란스럽지만 계속 보다보면 편리하면서도 많이 활용되는 함수라고 합니다. 그럽 한번 시작해보겠습니다!

REDUCE() 란?

  • 배열의 여러 연산을 할 때 간단하게 작성이 가능하도록 도와주는 함수

  • forEach() 보다 어떤 것을 구현하고자하는지 훨씬 명확하게 기능을 보여주는 함수

Reduce 함수 사용법

reduce는 2개의 매개변수를 가질 수 있어요. 처음에는 함수가 들어가고, 여기에는 화살표 함수가 사용가능하답니다. 두 번째 매개변수는 초기값이 들어가야해요. 혹시 초기값을 잊고 적지 않았다면, 배열의 첫 번째 값이 초기값으로 들어가게 됩니다. 이 경우, 루프를 돌 때 배열을 처음부터 돌지 않습니다!

이게 왜 문제가 될까요?

만약 배열에 값이 들어있지 않다면 에러가 나게되기때문입니다!! 그러니 초기값을 잊지말고 꼭 넣어주세요!

자, 그렇다면 함수에는 4개의 인자를 가지게됩니다. 

- accumulator : 누적된 값의 계산 (초기값이 들어갈 자리)

- current : 현재값 ( 배열에 들어있는 값을 가지고 루프를 돌 인자)

- index : 현재 배열에서의 인덱스

- array : 배열 그 자체

이렇게 4가지의 인자를 함수에서 사용할 수 있답니다.

자 그럼 한 번 예시를 보아요!

위의 예시는 4개의 인자를 다 사용한 경우입니다. 배열의 초기값을 0으로 잡아주어 acc의 초기값은 0이고 current는 배열의 0번째인 1부터 돌게 됩니다. 이 때 마지막까지 더하면 전체 배열의 길이만큼 나누어주어야 평균값이 계산 되겠죠? 그걸 표현한 함수가 바로 위의 예시가 됩니다.

reduce의 장점

  1. 앞서 말했다싶이 forEach보다 훨씬 명확하게 기능을 보여줍니다

  2. 초기값은 숫자뿐만 아니라 {}같은 배열,객체도 가능하고, 이런 결과를 그룹화하여 보여줄 수 있습니다.

이렇게 장점이 많다보니 자바스크립트에서 reduce는 꽤 많이 활용되고있답니다.

JavaScript: 비구조화 할당이란?

Posted on 2021-07-20

안녕하세요, 오늘은 자바스크립트를 훨씬 간단하게 작성하게 도와주는 비구조화 할당에 대해서 이야기를 나누어보려고 합니다.

비구조화 할당(destructuring assignment)이란?

  • 배열이나 객체 속성을 분해해서 그 값을 변수에 담게 해주는 표현

  • 객체 구조 분해라고도 부름

배열

만약 배열을 하나하나 변수로 지정해주고자 한다면 let user1 = users[0], let user2 = users[1]…이렇게 하나하나 작성해주어야 합니다. 만약, 비구조화 할당을 사용한다면 저렇게 일일이 적어줄 필요없이 한 줄로써 작성이 가능해 집니다.

변수를 하나하나 설정해주는 것 보다 훨씬 간단해졌죠?

만약 이런경우는 어떻게 해야할까요? let [a , b, c] = [10,20]

위의 경우는 undefined가c에 들어가게 됩니다. 하지만 undefined를 c에 넣고싶지 않다면, 간단하게 미리 기본값을 설정해줄 수 있습니다. 

```let [a=1,b=2,c=3] = [10,20]```

이렇게 미리 기본값을 설정해준다면 오류가 날 걱정도, undefined값이 들어갈일도 없어집니다. 또한, 기본값이 있다고 하더라도 설정해준 값이 우선시되며, 아무값도 넣지 않았을때만 기본값을 사용하게 됩니다.

여러분은 값을 서로 swap하고싶을때 어떻게 하시나요? 보통은 새로운 변수 c를 선언하여 값을 옮겨담죠. 하지만 구조분해할당을 사용한다면 [a,b] = [b,a] 한 줄로 swap이 가능하게 됩니다. 편리하죠?

객체

배열뿐만 아니라 객체도 비구조화 할당이 가능합니다. 배열과는 달리 객체의 경우 나열된 순서를 신경쓰지 않고 구조분해를 해도 됩니다.

이렇게 비구조화 할당을 하게 된다면 해당 값을 불러울때 user.name이렇게 하지 않고 바로 name으로 값을 불러올 수 있게 됩니다. 함수 내부에서도 사용이 가능해 많이 활용하게 됩니다.

객체는 key : 별명 의 형태로도 사용이 가능합니다. 즉, 새로운 변수 이름을 할당이 가능합니다.

이렇게 새로 할당된 변수 이름과 기존의 이름 모두 활용이 가능합니다. 위의 함수를 풀이해보자면, user의 name에서 값을 가지고와서 userName에 복사해서 넣어줘라는 뜻입니다. age도 마찬가지로 user의 age에서 값을 가지고와서 userAge에 넣음으로써 name과 userName 둘 다 같은 값을 가지게됩니다.

객체도 배열처럼 기본값 설정이 가능할까요?

정답은 ‘네’입니다. 객체도 배열처럼 기본값을 설정하여 사용할 수 있습니다.

또한, 깊숙한 곳에 있는 값을 꺼내올수도 있습니다.

이렇게한다면 훨씬 편리하게 사용이 가능하게 됩니다.

함수의 파라미터또한 객체의 비구조화 할당으로 사용이 가능하고, 모든 파라미터를 꼭 가져오지 않아도 됩니다.

이렇게 비구조화 할당에 대해서 알아보았는데 정리가 되시나요?

자바스크립트 : Getter & Setter

Posted on 2021-07-19

안녕하세요, 여러분!
오늘은 자바스크립트에서 getter와 setter에 대해서 알아보려고 합니다! 여러분들은 getter와 setter이 필요한 이유와 어떻게 사용하시는지 알고 계신가요? 모르신다면 이번 포스팅을 통해 함께 알아보아요.

프로퍼티

  1. 데이터 프로퍼티
  • 지금까지 사용한 (우리가 흔히 아는) 모든 프로퍼티
  1. 접근자 프로퍼티

  • 본질은 함수

  • 값을 얻고(get) 설정(set)하는 역할을 담당

GETTER와 SETTER이 필요한 이유

일반 데이터 프로퍼티를 사용하다보면 코드가 조금 지저분해보이고 길다고 생각해보신분 계신가요?

이 때 getter와 setter을 사용하면 됩니다. 접근자 프로퍼티를 사용하면 본질은 함수이지만 프로퍼티로 보이기 때문에 (사용하기 때문에) 코드길이가 조금 줄어들고, 외부에서 값을 받아 설정을 할 수 있답니다.

특징

  • get 함수와 set 함수의 이름은 동일해도 괜찮으며 실제 값은 별도의 프로퍼티 (_함수 이름)에 저장됩니다.

    • 예) get 함수 이름 : name() / 실제 값 : _name

  • get은 파라미터가 없는 함수로, 프로퍼티를 읽을 때 작동 (값을 얻어옴)

  • set은 파라미터가 하나인 함수로, 프로퍼티에 값을 쓸 때 호출됨 (값을 설정함)

이렇게 접근자 프로퍼티인 GETTER와 SETTER에 대해 알아보았는데 어떠신가요?

프로세스 실행구조 이해하기

Posted on 2021-07-18

Computer Processing

안녕하세요, 최근에 스택과 큐를 공부했는데 스택과 큐가 프로세스 실행 구조와 프로세스 스케쥴링 방>식에 많이 활용이 된다고 하더라구요. 그래서 오늘은 프로세스 실행 구조에 대해서 알아보려고 합니다.
자, 그러면 한번 출발해볼까요?

프로세스 실행구조

우선 프로세스에 대해 말하기 전, 작업(JOB)에 대해서 말해보겠습니다. 작업은 프로그램과 데이터를 말하는데요, 이런 작업을 하기 위해 커널(시스템)에 등록을 하게 됩니다. 커널에서 이러한 작업을
관리하는 이유는 시스템의 성능을 위해서입니다. 커널에 등록된 프로그램은 커널의 관리 하에 실행되>고, 각종 자원을 요청, 할당 및 반납할 수 있게 됩니다. 이 때, 프로세스라는 개념이 등장하게 됩니다. 프로세스는 자원을 할당받고 제어하게 되고, 이 때 프로세스 관리 블록이 생성됩니다.(프로세>스 관리 블록은 OS에서 프로세스 관리에 필요한 정보를 저장하는 것을 말합니다)

    *** 프로세서(processor) : CPU / 프로세스 : 실행중인 프로그램

프로세스의 메모리 구조

  1. 코드 영역 : 프로그램을 실행시키는 명령어 올라감 (자원)

  2. 데이터 영역 : 전역 변수, static 변수 (자원)

  3. 힙 영역 : 동적할당을 위한 메모리영역(자원)

  4. 스택 영역 : 지역 변수, 함수 호출 시 전달되는 파라미터를 위한 메모리 영역

     - 스레드 == 스레드 아이디 + 자원 + 스택 (자원은 공유하지만 제어는 달리함)

프로세스 상태

  1. Created State : 작업을 커널에 등록

     - 프로세스 생성 및 프로세스 관리 블록 할당

 - 메모리 공간이 있나 확인

  2. 메모리 공간이 있을 경우 (Active)

     - ready : 프로세서 외의 다른 모든 자원을 할당받은 상태 (즉시 실행 가능 상태)

 - running : 프로세서 + 필요한 모든 자원을 할당받은 상태

 - blobked / asleep : 프로세서 외의 다른 자원을 기다리는 상태

         - 서류를 잠시 빼놓고 왔을 경우 다시 가지러 간 상태라고 이해하기!

         - 여기서 running으로 바로갈 수는 없음. 다시 줄을 서서 기다려야 함(ready상태로 돌

    아감)

  3. 메모리 공간이 없을 경우 (Suspended)

     - 이전에 했던 작업을 swap device에 저장해두고 active 상태로 돌아갔을 때 복구해 사용

 - suspended ready : 프로세서가 없는 상태

 - suspended block : 프로세서와 기타 자원이 없는 상태

  4. Terminated / Zombie State : 프로세스 수행이 끝난 상태에 모든 자원을 반납후 커널 내에 일부 블>록 정보 (후에 활용할 수 있는 정보들)이 남아있는 상태 => 커널이 이 정보들을 수집

    스케줄링

스케줄링은 CPU(프로세서)하나일 때 동시에 여러개의 프로세스가 실행될 경우 할당 순서 및 방법을 결>정하는 것을 말합니다. 이런 스케줄링을 하는 이유는 CPU 이용률 증가, 균형있는 자원 사용, 처리율 증
가등의 이유로 실행하게 됩니다. 스케줄링 방식은 두 가지로 나뉘게 됩니다.

  1. 비선점 스케줄링

  • 프로세스에게 이미 할당된 CPU를 강제로 빼앗을 수 없고 사용이 끝날 때까지 기다려야하는 방법

  • CPU를 사용이 끝날때까지 넘겨주징나하 일괄 처리와 실시간 처리가 불가하다.

  • 오버헤드(과부하)는 일어나지않는다

  • 대표적인 스케줄링 : FIFO

  1. 선점 스케줄링

  • 우선 순위가 높은 다른 프로세스가 할당된 CPU를 강제로 빼앗을 수 있는 방법

  • 실시간 처리, 대화식 분할 처리 가능

  • 오버헤드가 일어날 가능성이 있음

FLOAT vs FLEXBOX vs GRID

Posted on 2021-07-17

안녕하세요, 여러분! 🙋

오늘은 float, flexbox, 그리고 grid를 알아보려고 합니다. 여러분들은 레이아웃을 잡을 때 어떤 속성을 사용하시나요? 흔히 float과 flexbox를 많이 사용하는데 그렇다면 이 둘의 차이점은 무엇일까요? 오늘은 float, flexbox 그리고 grid의 차이점에 대해서 적어보겠습니다!

FLOAT

Float은 이미지와 텍스트 배치를 위한(이미지를 텍스트가 감싸는 형태) 목적으로 나타났어요. 하지만 과거에는 배치 기능이 다양하지 않다보니 float을 활용해서 많이 사용했지만, 이제는 flexbox나 grid를 더 많이 사용하고 있어요.

특징

  • 여러개의 float을 가질 수 있다.

  • inline-block을 사용하여 배치하면 텍스트의 성질때문에 공백이 생겨버리지만, float은 상자 사이의 띄어쓰기(공백)이 사라졌다.

  • float은 블록 레이아웃의 사용을 뜻해 대부분 block으로 수정된다.

    • 단, flex와 inline-flex는 float 속성의 효과가 없다.

  • float속성이 적용된 요소의 주위는 다른 요소들이 흐르게되는등 의도치않은 상황이 발생하게 된다. (원래 가로배치를 위한 속성이 아니다)

    • 이 문제를 해결하기 위해서 clear(해제)를 해주어야한다.

FLEXBOX

Float의 문제점을 보완하고 나온 것이 바로 FLEXBOX인데요, FLOAT과는 달리 FLEXBOX는 행 또는 열을 만들기 위한 속성이예요. 자, 그렇다면 flexbox의 특징을 한 번 알아보아요!

특징

  • 행 또는 열을 (float에 비해) 쉽게 만들 수 있다.

    • 레이아웃 : 행 and 열 / flexbox : 행 or 열

  • 요소의 크기가 불분명하거나 동적인 경우에도 각 요소를 정렬할 수 있는 효율적인 방법을 제공한다.

  • Container과 items라는 개념으로 이루어져있다.

    • container은 부모 요소이고, items는 자식 요소이다.

    • container은 필수 요소이다.

    • flexbox를 사용하기 위해 container에 “display:flex”를 적용시켜준다.

GRID

자, flexbox의 특징들을 보니 훨씬 가로 배치에 특화되어있음을 알 수 있죠? 그러면 grid는 언제 사용하나요?

Grid는 flexbox와 float의 단점을 보완하고 나온 가장 최신 레이아웃 속성이예요. 제가 지금 레이아웃이라고 한 이유는 grid는 행과 열을 위한 목적으로 만들어졌기 때문이죠. 바로 이 점이 flexbox와 차이점이랍니다! flex box는 행 또는 열 (가로 배치에 조금 더 많이 사용해요)이지만 grid는 행 그리고 열을 만들어 관리할 수 있어요.

특징

  • 레이아웃을 위해서 만들어졌다.

  • 각 상자마다 사이즈를 달리할 수 있고 간격도 설정이 가능하다

  • 다만, 브라우저에 제약사항이 있다 (모든 브라우저가 지원하는 기술은 아니다)

이렇게 오늘은 float, flexbox, grid에 대해서 알아보았는데요, 현재는 float보다는 flexbox나 gird를 더 많이 사용하는 추세라고해요.

JavaScript : 함수와 화살표 함수의 차이점

Posted on 2021-07-16

안녕하세요:))

오늘은 ES6에 추가된 ‘화살표 함수’에 대해서 알아보도록 하겠습니다!
화살표 함수(Arrow Function)는 일반 함수와는 달리 간략히 적을수 있다는 장점을 가졌는데요, 이 화살표 함수의 특징은 무엇인지, 그리고 일반함수와의 차이는 무엇인지 알아보도록 하겠습니다!

화살표 함수(Arrow Function)이란?

  • 화살표 함수는 익명 함수로만 사용이 가능합니다.

  • 일반함수와 달리 function을 적지 않고 ()=> 만으로 함수를 선언할 수 있습니다.

  • 매개변수가 1개인 경우, 소괄호 생략이 가능합니다.

  • 중괄호 생략 또한 가능하고, 자동으로 값을 return 합니다.

  • 매개변수에 default값도 적용이 가능합니다.

  • 그래서 아주 간결하게 값을 받고 반환할 수 있습니다. (간략하게 적을 수 있을만큼 줄여서 완전 편리하죠!)

새로 추가된 문법인 만큼 더 간편해졌죠! 이쯤되면 🙋’아니, 그럼 함수대신 화살표 함수를 사용하면 되지, 왜 함수를 사용할까?’ 라는 생각이 드시는 분도 있을거예요. 자, 그러면 이제 화살표 함수와 함수의 차이를 알아볼까요?

화살표 함수 vs 일반 함수

  1. this

    • 일반 함수에서 this는 동적으로 바인딩하지만, 화살표 함수에서는 this를 바인딩하지 않습니다. 그래서 화살표 함수는 객체안에 함수를 넣을 때 등 사용에 제약사항이 있습니다.

  2. 생성자 함수

    • 일반함수와 달리 화살표 함수는 프로토타입 프로퍼티를 가지고 있지 않아 생성자 함수로 사용할 수 없습니다.

      • 프로토타입 프로퍼티란?

      JS는 클래스라는 개념이 없다보니 기존 객체를 복사하여 새로운 객체를 생성하는 프로토타입 기반의 언어입니다. 여기서 프로토타입 프로퍼티를 프로토타입 객체를 참조하는 프로토타입 속성이라고 이해하면 됩니다.
